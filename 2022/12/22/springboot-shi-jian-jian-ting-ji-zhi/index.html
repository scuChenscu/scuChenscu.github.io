SpringBoot事件监听机制

Java SpringBoot
技术 研发
发布日期:   2022-12-22
文章字数:   833
阅读时长:   3 分

Spring事件机制关键对象

Spring事件机制主要涉及4个关键对象：事件、监听器、发布者、管理者

ApplicationEvent：事件对象

public abstract class ApplicationEvent extends EventObject { // 继承JDK的EventObject类
    private static final long serialVersionUID = 7099057708183571937L;
    private final long timestamp; // 可以携带事件的时间戳

    public ApplicationEvent(Object source) {
        super(source);
        this.timestamp = System.currentTimeMillis();
    }

    public ApplicationEvent(Object source, Clock clock) {
        super(source);
        this.timestamp = clock.millis();
    }

    public final long getTimestamp() {
        return this.timestamp;
    }
}

ApplicationListener：事件监听器

@FunctionalInterface // 函数式接口
public interface ApplicationListener<E extends ApplicationEvent> extends EventListener { // 继承JDK的EventListener接口
    void onApplicationEvent(E event);

    static <T> ApplicationListener<PayloadApplicationEvent<T>> forPayload(Consumer<T> consumer) {
        return (event) -> {
            consumer.accept(event.getPayload());
        };
    }
}

ApplicationEventPublisher：事件发布者

封装了事件有关的公共方法，作为ApplicationContext的超级街廓，也是委托ApplicationEventMulticaster完成事件发布。

ApplicationEventMulticaster：事件广播器

管理监听器和发布事件，ApplicationContext通过委托ApplicationEventMulticaster来发布事件

示例

定义事件

通过继承ApplicationEvent定义事件

public class MyEvent extends ApplicationEvent {
    public MyEvent(Object source) {
        super(source);
    }

    public MyEvent(Object source, Clock clock) {
        super(source, clock);
    }
}

定义监听器

监听器负责监听事件，并进行相关的逻辑处理；这里有两种方式实现：

  • 实现ApplicationListener接口

    @Component
public class MyListener implements ApplicationListener<MyEvent> {
	@Async // 通常使用异步，不然如果有多个监听器监听同一个事件，会导致阻塞
    @Override
    public void onApplicationEvent(MyEvent event) {
        // 逻辑处理
        System.out.println("event");
    }
}

  • 在方法上使用@EventListener注解

@Component
public class MyListener {
	@Async
    @EventListener
    public void handleEvent(MyEvent event) {
        // 逻辑处理
        System.out.println("event");
    }
}

发布事件

@Service
public class MyService {
    @Autowired
    private ApplicationEventPublisher applicationEventPublisher;

    public void method() {
        applicationEventPublisher.publishEvent(new MyEvent(this));
    }
}

原理

核心就是SpringBoot的ApplicationEventMulticaster对发布者和监听器进行管理，当发布者发布事件时，通知监听器。

初始化

SpringBoot在启动时会refreshContext；refreshContext中调用refresh()方法；会在refresh()方法进行相关的初始化工作：

this.initApplicationEventMulticaster(); // 初始化事件广播器
this.registerListeners(); // 注册监听器
// 注册监听器：事件广播器维护一个监听器的集合，注册监听器就是把监听器加入集合
 this.getApplicationEventMulticaster().addApplicationListener(listener);

发布事件

当service层通过applicationEventPublisher发布事件后，ApplicationEventMulticaster接口的子类SimpleApplicationEventMulticaster负责广播事件：

public void multicastEvent(final ApplicationEvent event, @Nullable ResolvableType eventType) {
    ResolvableType type = eventType != null ? eventType : this.resolveDefaultEventType(event);
    Executor executor = this.getTaskExecutor();
    Iterator var5 = this.getApplicationListeners(event, type).iterator(); // 根据事件获取对应的广播器集合，进行迭代遍历

    while(var5.hasNext()) {
        ApplicationListener<?> listener = (ApplicationListener)var5.next();
        if (executor != null) {
            executor.execute(() -> {
                this.invokeListener(listener, event);
            });
        } else {
            this.invokeListener(listener, event);
        }
    }

}

this.invokeListener()方法实际上就是：

listener.onApplicationEvent(event);

注意，监听器默认是串行执行，想要异步执行，可以给广播器指定线程池：executor != null；或者使用@Async注解。

Async注解

Spring容器启动初始化bean时，判断类中是否使用了@Async注解，创建切入点和切入点处理器，根据切入点创建代理，在调用@Async注解标注的方法时，会调用代理，执行切入点处理器invoke方法，将方法的执行提交给线程池，实现异步执行。所以，需要注意的一个错误用法是，如果A类的a方法(没有标注@Async)调用它自己的b方法(标注@Async)是不会异步执行的，因为从a方法进入调用的都是它本身，不会进入代理。
