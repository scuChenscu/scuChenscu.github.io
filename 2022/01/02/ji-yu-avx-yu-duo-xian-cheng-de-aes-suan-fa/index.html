基于AVX与多线程的AES算法

摘要

AES加一种常见的对称加密算法，常用于网络数据加密传输；AVX是一种单指令流多数据流的指令集，常用于加速计算；多线程技术可以充分利用CPU资源，提高程序的执行效率。本项目将基于AVX与多线程实现AES算法：使用C语言调用AVX指令集实现AES算法，使用Go语言创建多线程调用C代码实现分组加密；并进行了原生的、使用AVX加速的、使用多线程加速的、同时使用AVX和多线程加速的AES算法加密性能比较。

AVX指令集

高级向量扩展指令集（Advanced Vector Extensions，AVX）是x86架构微处理器中的单指令流多数据流（Single Instruction Stream, Multiple Data Stream，SIMD）指令集；即可以实现一次指令操作多组数据，如下图所示。

AVX的应用场景

  • 渲染、视频编码：Blender三维图形图像软件的渲染引擎可以调用AVX2指令集进行加速计算制作动画或者CG图；Intel开源的视频编码器SVT，使用AVX配合不同后端可以实现不同的编码；
  • 深度学习：Google著名的开源深度学习框架Tensorflow在1.6版本之后应用了AVX指令集；英特尔也针对深度学习设计了AVX-512 VNNI指令集，用来加速深度学习相关的计算；
  • 加解密：libsodium加密库提供了从AVX到AVX-512的优化，Linux内核也支持使用AVX和AVX2指令集进行加解密计算，还会配合AES-NI这个专用的指令集；
  • 游戏：一组由英特尔研究院开发的高性能光线追踪内核Embree高度依赖AVX指令集，可在最新的英特尔® 处理器上进行照片级的渲染。

AES加密算法

AES简介

高级加密标准（Advanced Encryption Standard，AES）是最常见的对称加密算法之一，例如微信小程序就使用该算法进行数据加密传输。

对称加密与非对称加密

对称加密：加密与解密使用相同的密钥；优点是加解密速度非常快，适用于频繁传输数据的场合；缺点是传输密钥的安全性难以保证；常见的对称加密算法有DES、AES等。

非对称加密：密钥分为公钥和私钥；公钥加密的数据使用私钥解密；私钥加密的数据使用公钥解密；优点是密钥传输非常方便，不需要当心安全性；缺点是加解密速度非常慢，仅适用偶尔传输数据的场合；常见的非堆成加密算法有RSA、ECC等。

在实际应用中，通常是先使用非对称加密算法RSA和加密对称加密算法AES密钥发送给接收方，接收方使用RSA公钥解密得到AES密钥，然后双方使用AES算法加密数据传输通信（AES+RSA混合加密，如下图所示）。

AES的基本结构

AES为分组密码，它把明文分成多个长度相等的组，分别对每组数据进行加密，加密结果拼接得到密文，如下图所示。在AES标准规范中，分组长度只能是128个比特，即每个分组为16个字节。密钥的长度可以使用128位、192位或256位。本文主要是实现AES-128，也就是密钥的长度为128位，加密轮数为10轮。

AES算法加密步骤

AES算法在加密之前，会先使用初始密钥进行密钥扩展得到轮子密钥，轮子密钥将会用在对应轮次的加密；密钥扩展过程如下图所示：

  • 第一步，将种子密钥按上图的格式排列，其中依次表示种子密钥的一个字节；排列后用4个32比特的字表示，分别记为；

  • 第二步，由得到，计算方式如下：

    // 密钥扩展
void aes_key_expansion(uint8_t *key, uint8_t *w)
{

    uint8_t tmp[4];
    uint8_t i;
    uint8_t len = Nb * (Nr + 1);

    for (i = 0; i < Nk; i++)
    {
        w[4 * i + 0] = key[4 * i + 0];
        w[4 * i + 1] = key[4 * i + 1];
        w[4 * i + 2] = key[4 * i + 2];
        w[4 * i + 3] = key[4 * i + 3];
    }

    for (i = Nk; i < len; i++)
    {
        tmp[0] = w[4 * (i - 1) + 0];
        tmp[1] = w[4 * (i - 1) + 1];
        tmp[2] = w[4 * (i - 1) + 2];
        tmp[3] = w[4 * (i - 1) + 3];

        if (i % Nk == 0)
        {   // g函数
            rot_word(tmp);
            sub_word(tmp);
            coef_add(tmp, Rcon(i / Nk), tmp);
        }
        else if (Nk > 6 && i % Nk == 4)
        {
            sub_word(tmp);
        }

        w[4 * i + 0] = w[4 * (i - Nk) + 0] ^ tmp[0];
        w[4 * i + 1] = w[4 * (i - Nk) + 1] ^ tmp[1];
        w[4 * i + 2] = w[4 * (i - Nk) + 2] ^ tmp[2];
        w[4 * i + 3] = w[4 * (i - Nk) + 3] ^ tmp[3];
    }
}

其中，密钥扩展使用的g函数实现过程如下图所示：第一步，将循环左移8比特；第二步，分别对每个字节借助进行字节代换，得到新的字节；第三步，与32比特的常量进行异或操作，得到；

完成密钥扩展后，开始正式加密，AES加密主要包含以下四种操作：字节代换、行移位、列混合和轮密钥加；前9轮加密都包含了四种操作，最后一轮加密不包含列混合操作；明文经过这10轮加密后就得到了最终的密文。加密过程如下图所示：

/**
 * @brief AES加密的主函数
 * @param in 输入明文
 * @param out 输出密文
 * @param w 密钥
 */
void avx_aes_encode(uint8_t *in, uint8_t *out, uint8_t *w)
{
    uint8_t r = 0;

    // 初始化，将数组数据装入__m256i
    __m256i avx_state = avx_set_data_u8(in);
  	// 设置当前轮密钥
    __m256i avx_round_key = avx_set_round_key(r, w);

    // 1. 轮密钥加
    __m256i avx_state1 = avx_add_round_key(avx_state, avx_round_key);

    // 2. 循环前9轮加密
    for (r = 1; r < 10; ++r)
    {
        avx_round_key = avx_set_round_key(r, w);
        avx_state1 = avx_aes_loop(avx_state1, avx_round_key);
    }

    // 3. 最后1轮加密
    avx_round_key = avx_set_round_key(r, w);
    __m256i avx_state_final = avx_aes_final(avx_state1, avx_round_key);

    // 4.输出密文
    get_avx_output(&avx_state_final, out);
}

// 前9轮加密：字节代换->行移位->列混合->轮密钥加
__m256i avx_aes_loop(__m256i state, __m256i round_key)
{
    return avx_add_round_key(
        avx_mix_column(
            avx_shift_rows(
                avx_sub_bytes(state))),
        round_key);
}

// 最后1轮加密：字节代换->行移位->轮密钥加
__m256i avx_aes_final(__m256i state, __m256i round_key)
{
    return avx_add_round_key(
        avx_shift_rows(
            avx_sub_bytes(state)),
        round_key);
}

    轮密钥加是将输入与该轮子密钥进行异或操作；

  2. 字节代换

    字节代换是借助完成一个字节到另外一个字节的映射；用于提供密码算法的混淆性。映射规则：输入字节的高4位作为行标，低4位作为列标，得到新字节；假设输入字节的值为，则输出值为；具体地，字节0x经过字节代换得到：0x

    // 字节代换
__m256i avx_sub_bytes(__m256i in_state)
{
    alignas(32) uint16_t temp_state[16];
    _mm256_store_si256((__m256i *)&temp_state, in_state);
    alignas(32) uint16_t state[16];

    for (uint8_t i = 0; i < 4; ++i)
        for (uint8_t j = 0; j < 4; ++j)
          	// 查S_Box，进行字节代换
            state[4 * i + j] = (uint16_t)getSBoxValue(temp_state[4 * i + j]);

    __m256i res = avx_set_data_u16(state);

    return res;
}

  3. 行移位

    行移位是一个4x4的矩阵内部字节之间的置换，用于提供算法的扩散性。主要步骤：第一行保持不变，第二行循环左移8比特，第三行循环左移16比特，第四行循环左移24比特；

    AVX实现行移位：

    // 行移位
__m256i avx_shift_rows(__m256i input)
{
    __m256i state = input;
    __m256i left = _mm256_set_epi64x(left1[3], left1[2], left1[1], left1[0]);
    __m256i right = _mm256_set_epi64x(right1[3], right1[2], right1[1], right1[0]);
    __m256i shift_left = _mm256_sllv_epi64(input, left);
    __m256i shift_right = _mm256_srlv_epi64(input, right);
    __m256i res = _mm256_or_si256(shift_left, shift_right);

    return res;
}

/**
 * @brief avx_mix_column 列混合函数，将state分为四组进行列混合计算，然后将结果合并
 * @param state 
 * @return __m256i 
 */
__m256i avx_mix_column(__m256i state)
{
    // 将要计算的矩阵值分为四组，依次左移到对角线计算
    __m256i state_l0 = state;
    __m256i state_l1 = avx_left_shift_step(state_l0);
    __m256i state_l2 = avx_left_shift_step(state_l1);
    __m256i state_l3 = avx_left_shift_step(state_l2);
    // 分别用调辅助函数，计算有限域乘法的结果
    __m256i raw_res_part1 = avx_mix_colomn_add_helper(state_l0);
    __m256i raw_res_part2 = avx_mix_colomn_add_helper(state_l1);
    __m256i raw_res_part3 = avx_mix_colomn_add_helper(state_l2);
    __m256i raw_res_part4 = avx_mix_colomn_add_helper(state_l3);
		// 结果汇合输出
    __m256i res = _mm256_or_si256(raw_res_part1,
                                  _mm256_or_si256(avx_right_shift_step(raw_res_part2),
                                                  _mm256_or_si256(avx_right_shift_step(avx_right_shift_step(raw_res_part3)),
                                                                  avx_right_shift_step(avx_right_shift_step(avx_right_shift_step(raw_res_part4))))));

    return res;
}

 * @brief 列混合辅助函数，实现了两矩阵之间对应位置的基于有限域的加法
 * @param state     
 * @return __m256i 
 */
__m256i avx_mix_colomn_add_helper(__m256i state)
{		
  	// T矩阵装入__m256i
    __m256i avx_T = avx_set_data_u16(p_T);
    // T矩阵移位
    __m256i tl0 = avx_T;
    __m256i tl1 = avx_left_shift_step(tl0);
    __m256i tl2 = avx_left_shift_step(tl1);
    __m256i tl3 = avx_left_shift_step(tl2);
    // state移位
    __m256i su0 = state;
    __m256i su1 = avx_up_shift_step(su0);
    __m256i su2 = avx_up_shift_step(su1);
    __m256i su3 = avx_up_shift_step(su2);
    // 取出对角线元素，做有限域乘法运算
    __m256i mul0 = avx_mix_colomn_helper(avx_select_diag(su0), avx_select_diag(tl0));
    __m256i mul1 = avx_mix_colomn_helper(avx_select_diag(su1), avx_select_diag(tl1));
    __m256i mul2 = avx_mix_colomn_helper(avx_select_diag(su2), avx_select_diag(tl2));
    __m256i mul3 = avx_mix_colomn_helper(avx_select_diag(su3), avx_select_diag(tl3));
    // 汇合结果，对做完有限域乘法的结果进行有限域加法计算
    __m256i res1 = _mm256_xor_si256(mul2, mul3);
    __m256i res2 = _mm256_xor_si256(mul0, mul1);
    __m256i res = _mm256_xor_si256(res1, res2);

    return res;
}

/**
 * @brief 列混合辅助函数，实现了两矩阵之间对应位置的基于有限域的乘法
 * @param state     输入状态
 * @param TT        T矩阵
 * @return __m256i 
 */
__m256i avx_mix_colomn_helper(__m256i state, __m256i TT)
{
    __m256i avx_mask_bit = avx_set_data_u16(p_mask_bit);
    __m256i avx_mask_int16 = avx_set_data_u16(p_mask_int16);
    __m256i avx_multiplier = avx_set_data_u16(p_multiplier);
  
    __m256i state_mul_t_and_mask = _mm256_mullo_epi16(state, _mm256_and_si256(TT, avx_mask_bit));

  	__m256i s_right_7 = _mm256_srli_epi16(state, 7);
    __m256i t_right_1 = _mm256_srli_epi16(TT, 1);
    __m256i sr7_and_tr1 = _mm256_and_si256(s_right_7, t_right_1);
    __m256i sr7_and_tr1_mul_0x1b = _mm256_mullo_epi16(sr7_and_tr1, avx_multiplier);
    
    __m256i s_left_1 = _mm256_and_si256(avx_mask_int16, _mm256_slli_epi16(state, 1));
    __m256i s_left_1_mul_t_right_1 = _mm256_mullo_epi16(s_left_1, t_right_1);
    
    __m256i res = _mm256_xor_si256(state_mul_t_and_mask,
                                   _mm256_xor_si256(s_left_1_mul_t_right_1,
                                                    sr7_and_tr1_mul_0x1b));

    return res;
}

多线程

多线程指从软件或者硬件上实现多个线程并发执行的技术，可以充分发挥计算机的性能。在多线程程序中，当一个线程需要等待资源，CPU可以切换执行其他线程，大大提高了程序的效率。

Go协程调用C代码

AES为分组密码，因为每个分组的加密过程是彼此独立的，所以可以采用多线程技术对每个分组进行加密，再将分组加密结果合并成最终的密文；

// 定义管道中的消息的结构体
type message struct {
	data []byte		// 数据
	id   int			// 编号，用于对goroutine乱序发到channel中的数据进行重新排序
}

// encode函数将拿到的16字节数据通过调用C语言实现的函数实现加密，并将处理完的结果发回到channel 
func encode(msg []byte, waitgroup *sync.WaitGroup, buff chan<- message, i int) {
	defer func() {
		if err := recover(); err != nil {
			fmt.Printf("Panicing in encode: %s\r\n", err)
		}
	}()
	d := (*C.uint8_t)(unsafe.Pointer(&msg[0]))
	a := C.run(d)
  
	// reference https://github.com/golang/go/issues/13656#issuecomment-165867188
	sh := reflect.SliceHeader{uintptr(unsafe.Pointer(a)), 16, 32}
	out := *(*[]C.uint8_t)(unsafe.Pointer(&sh))
	var outslice []byte
	for _, d := range out {
		outslice = append(outslice, byte(d))
	}
	var temp message
	temp.data = outslice
	temp.id = i
	buff <- temp
	waitgroup.Done()
}

// Encode函数对接收到的任意长度字符串做末尾填充后分段
// 调用新开goroutine来执行对每个分段的编码，等待至所有goroutine完成后收集结果并排序后拼接返回
func Encode(input string) (res []byte) {
	defer func() {
		if err := recover(); err != nil {
			fmt.Printf("Panicing %s\r\n", err)
		}
	}()
	var rawResult []message

	wg := sync.WaitGroup{}
	channel := make(chan message, 10000)

	padtext := PaddingByte([]byte(input))
	var data [][]byte

	for len(padtext) > 0 {
		if len(padtext)%16 != 0 {
			fmt.Errorf("wrong length!")
		}
		data = append(data, padtext[:16])
		padtext = padtext[16:]
	}

	for index, str128 := range data {
		wg.Add(1)
		go encode(str128, &wg, channel, index)
	}

	wg.Wait()
	close(channel)

	for aa := range channel {
		rawResult = append(rawResult, aa)
	}
	// 收到的结果进行排序
	sort.Slice(rawResult, func(i, j int) bool {
		return rawResult[i].id < rawResult[j].id
	})

	for len(rawResult) > 0 {
		res = append(res, rawResult[0].data...)
		rawResult = rawResult[1:]
	}
	return
}

注意到，明文分组中可能会出现最后一个分组不满足16个字节的硬性要求，对此，我们使用以下方式处理：

  • 当明文满足要求，即message % 16 == 0，填充一个新的分组，即填充16个字节的0；
  • 当明文不满足要求，即message % 16 == n，n>0，在最后一个分组填充n至16个字节；
// PaddingByte 在cipherText末尾进行填充，填充后长度为16的整数倍
func PaddingByte(cipherText []byte) []byte {
	return padding(cipherText, 16)
}
// 计算需填充的长度，并按规则填充
func padding(cipherText []byte, blockSize int) []byte {
	padding := blockSize - len(cipherText)%blockSize
	if padding == 0 {
		return append(cipherText, bytes.Repeat([]byte{byte(padding)}, 16)...)
	}
	padText := bytes.Repeat([]byte{byte(padding)}, padding)
	return append(cipherText, padText...)
}

性能比较

实验环境：

针对不同的明文长度，分别对原生（Basic）、多线程加速（Multithreading）、AVX加速（AVX）和同时使用AVX和多线程加速（Multithreading+AVX）四种方式的AES加密算法多次重复实验，最终取平均值，进行性能比较，实验结果如下图所示：

实验结果分析：

  • 在明文长度小于10000时，Multithreading和Multithreading+AVX的性能反而是不如Basic，考虑是数据量较小的时候，在整个算法实现的耗时上，开线程的耗时占的比重要高于加密耗时，所以没有体现出多线程的性能优势；
  • 随着明文长度的增加，Multithreading和Multithreading+AVX的性能逐渐开始优于Basic，并且整个曲线增长较为平缓，而Basic则是倾向线性增长的趋势；
  • 从整个过程来看，AVX的性能都要优于Basic，说明AVX对整个AES加密算法的计算过程起到了加速效果；
  • 从整体上看，Multithreading和AVX性能最优，彼此接近；对于CPU计算密集型的任务，多线程是并行多个任务，而AVX可以理解为更加细粒度的并行，即单个任务内部的计算并行；
  • 从整体上看，Multithreading+AVX的性能是不如单独的Multithreading或者AVX，认为是受限于系统资源，AES加密主要是CPU计算、对寄存器的使用，几乎没有IO调度，所以同时使用Multithreading+AVX会增加资源的争抢，反而导致了性能降低。

AES算法解密步骤

AES解密是加密的逆运算，解密过程与加密较为类似，故此处省略源代码分析。

  1. 字节代换
